1.

(10) Define the 'term variable' in the context of an app inventor app.

Answer
A variable is a named memory cell, a place where information can be stored.

2.

(10) In the PaintPot app shown on the other page a variable is needed for the size of the circles but not for the size of the lines. Explain.

Answer
Canvas has a property, lineSize, that specifies the width of lines that are drawn. There is no such property for circle size, so you must define a variable to keep track of it and whether the user has chosen big (2 pixels) or small (8 pixels). In general, define a variable when you need to remember some information that is not already defined as a component property.

3.

(10) In App Inventor, how are a variable and a property similar? How are they different?

Answer
Component properties and variables are both "named memory cells". They are different in that a property is part of a component, whereas a variable is a "free-agent" memory cell not associated with a particular component. So properties are named in the form < componentName >.< propertyName >, e.g., Canvas.Width, while variables are just a single name, e.g., dotsize.
